# EUC Evolution Summary

This document summarises the evolution of End User Computing.

## 1960s–1970s: Mainframe Era

Architecture:
Centralised computing with dumb terminals connected to a mainframe.

Characteristics:

- centralised compute
- batch processing
- minimal user autonomy
- high security through centralisation

Key Vendors:

- IBM
- DEC
- Honeywell

Key Insight:
Modern cloud desktops replicate this architecture with remote compute.

---

## 1980s: Personal Computer Revolution

Architecture:
Local computing on individual desktops.

Characteristics:

- decentralised computing
- local applications
- limited IT control
- rapid software innovation

Key Vendors:

- Microsoft
- Apple
- IBM

Key Insight:
User freedom increased dramatically, creating new management challenges.

---

## 1990s: Client‑Server & Citrix

Architecture:

```
Client PC
   │
Application Server
   │
Database
```

Key Technologies:

- Citrix ICA
- Microsoft Terminal Services
- LAN networking

Impact:
Applications centralised again while PCs remained endpoints.

---

## 2000s: Virtualisation & VDI

Architecture:

```
Endpoint
   │
Broker
   │
Hypervisor
   │
Virtual Desktop
```

Key Vendors:

- VMware
- Citrix
- Microsoft

Challenges:

- storage performance
- infrastructure complexity
- profile management

---

## 2010s: Cloud & SaaS

Architecture:

```
User
   │
Internet
   │
SaaS / Cloud Services
```

Major Trends:

- SaaS adoption
- identity‑centric security
- mobile devices
- SD‑WAN networking

---

## 2020s: Cloud Desktop Era

Architecture:

```
User Device
   │
Identity / Security
   │
Cloud Desktop Platform
   │
Applications
```

Platforms:

- Azure Virtual Desktop
- Windows 365
- Citrix Cloud

Key Trend:
Operational complexity shifts from infrastructure to automation.
